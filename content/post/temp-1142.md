<p>jQuery CompositeUI</p>  <p>XD concerns, rebase links, info</p>  <p>browser quirks (e.g. jTemplates, eager loading of images, etc.)</p>  <p>jTemplates</p>  <p>jQuery hijaxing/form posts</p>  <p>Client-side stitching will ultimately become easier as modern, xd-capable browsers become more widely adopted.  But for now you can easily compensate by creating multiple virtual appilcations under your root IIS/apache web application, e.g. /apps/service.bcname/</p>  <p>Each virtual application can run in its own process and does not inherit any of its parents web.config, etc. essentially it becomes a completely isolated app, except that it has access to the cookies set by the root web application.</p>  <p>If we wanted, we could put an ngnix server in to proxy the requests and then it could re-write requests to /apps/whatever to whatever.website.com—this would be a temporary work-around if we really wanted cross-domain stuff but wanted to support older browsers</p>  <p>Root will host most of the common JS and styles/images, etc.</p>  <p>When developing the child app, it should have a copy (in some other repo) of the root scripts and possibly styles?</p>
