<p>I try to avoid the debugger at all costs, but it does come in handy sometimes.&# 160; You’ll even catch Uncle Bob in the debugger on occasion.&# 160; When using a compiled language, you basically “lose” the source code once compiled.&# 160; Of course bytecode languages can be decompiled, but that’s beside the point.</p> <p>We have a number of common library projects that we share between our applications.&# 160; Usually these common projects are very low-level infrastructure concerns, etc.&# 160; While these common projects are relatively mature, they do exhibit bugs from time to time.&# 160; Wouldn’t it be nice to step in to the source code of one of those projects while debugging your application?</p> <p>Enter Microsoft Source Server. It has the responsibility of “indexing” your source code.&# 160; Specifically, it hooks the PDB files back to the source code, but enables Visual Studio to pull the source from a source control repository, such as Subversion or TFS.&# 160; That’s exactly what we need.&# 160; In fact, at my company, we’ve been using this very solution for several years with great success.&# 160; As part of our build process, we would index the PDB files to facilitate runtime debugging of common/shared projects when referenced in another project while using Subversion for source control.</p> <p>Enter git.&# 160; Git is relatively new in the source control world—only about four years old as of this writing.&# 160; It’s even newer in the Windows world with very little tooling support, although that seems to be changing rapidly.&# 160; Because of this, there is no “Git adapter” to use with Microsoft Source Server.</p> <p>I looked around the internet for anyone who had written anything related to Git and Microsoft Source Server.&# 160; I found <a href="http://www.imaginaryrealities.com/post/2009/05/03/Indexing-source-code-in-Git-with-Source-Server.aspx">one article</a>, which turned out to be exactly what I needed.&# 160; Well, almost.&# 160; In the end, I completely re-wrote the implementation to be a little bit more “Clean Code” and to support distributed workflows better, even though I’m not a Perl programmer.&# 160; The end result was <a href="http://github.com/joliver/SourceServer-GitExtensions/tree/master">SourceServer-GitExtensions</a>.</p> <p>I have written a complete start-to-finish <a href="http://github.com/joliver/SourceServer-GitExtensions/raw/master/Readme.pdf">implementation guide</a> on how to setup indexing with Git and how to get debugging working in Visual Studio.&# 160; The source code for the project can <a href="http://github.com/joliver/SourceServer-GitExtensions">be found here</a>.</p>
