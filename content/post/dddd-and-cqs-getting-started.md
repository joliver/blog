<p>There are a number of good resources for getting started on Distributed Domain-Driven Design as well as architectural-level Command Query Separation (not to be confused with CQS as a programming paradigm).</p> <p><strong>Basic Concepts</strong></p> <p>For starters, you should be familiar with &quot;regular&quot; DDD: <a href="http://en.wikipedia.org/wiki/Domain-driven_design">Domain-Driven Design</a>, <a href="http://www.infoq.com/news/2006/12/domain-driven-design">DDD Quickly</a>, and <a href="http://thinkddd.com/">Think DDD</a>.</p> <p>Then, you'll probably want to understand about distributed systems because the way that you think about programming in a distributed environment is most definitely not the traditional &quot;request/response&quot; paradigm: <a href="http://www.amazon.com/Enterprise-Integration-Patterns-Designing-Addison-Wesley/dp/0321200683/ref=pd_sim_b_2">Enterprise Integration Patterns</a>, <a href="http://www.amazon.com/Pattern-Oriented-Software-Architecture-Distributed-Computing/dp/0470059028">Pattern-Oriented Software Architecture, Volume 4 (Distributed Computing)</a>, and <a href="http://www.amazon.com/Distributed-Event-Based-Systems-Gero-M&uuml;hl/dp/3540326510/ref=sr_1_1?ie=UTF8&amp;s=books&amp;qid=1208908666&amp;sr=1-1">Distributed, Event-Based Systems</a>.</p> <p>From there, you'll want to read up just a bit on <a href="http://www.udidahan.com/2008/08/11/command-query-separation-and-soa/">Command Query Separation and SOA</a> on Udi Dahan's blog as well as some of the <a href="http://nservicebus.com/Overview.aspx">processing methods</a> using messaging.</p> <p><strong>Videos & Screencasts</strong></p> <p>You may want to allocate a block of time. I can promise you it is probably the best few hours you'll ever spend. Think of this as an investment. If you spend all day writing code, wouldn't it be nice to &quot;sharpen the saw&quot; from time to time?</p> <p>With that block of time there are several videos that are worth watching. I advocate watching them in order listed below. I had to watch each several times to really have things sink in.</p> <p><a href="http://codebetter.com/blogs/gregyoung/archive/2008/05/20/devteach-talk.aspx">devTeach</a> - The best &quot;primer&quot; to DDDD and CQS</p> <p><a href="http://vimeo.com/3171910">European Virtual ALT.NET Screencast</a> - The quality of the video isn't the best, but it expands upon the original concepts in the devTeach video.</p> <p><a href="http://www.infoq.com/interviews/Architecture-Eric-Evans-Interviews-Greg-Young">Eric Evans interviews Greg Young</a></p> <p><a href="http://www.infoq.com/interviews/greg-young-ddd">Greg Young discusses State Transitions in DDD</a></p> <p><a href="http://codebetter.com/blogs/gregyoung/archive/2008/04/25/altdotnet-seattle-session-video.aspx">Seattle ALT.NET Meeting</a></p> <p><strong>Follow-up</strong></p> <p>Once you're done watching those videos, you'll be able to better understand <a href="http://codebetter.com/blogs/gregyoung/archive/tags/DDDD/default.aspx">Greg Young's blog posts</a>.</p> <p>In addition, there are a number of other really good resources:</p> <p><a title="http://www.bjoernrochel.de/2008/11/20/command-query-separation/" href="http://www.bjoernrochel.de/2008/11/20/command-query-separation/">http://www.bjoernrochel.de/2008/11/20/command-query-separation/</a></p> <p><a title="http://www.goeleven.com/blog/EntryDetail.aspx?entry=133" href="http://www.goeleven.com/blog/EntryDetail.aspx?entry=133">http://www.goeleven.com/blog/EntryDetail.aspx?entry=133</a></p> <p><a href="http://blog.xebia.com/2008/11/22/qcon-san-francisco-2008-unleash-your-domain/">http://blog.xebia.com/2008/11/22/qcon-san-francisco-2008-unleash-your-domain/</a></p> <p><a title="http://elegantcode.com/2008/04/30/altnet-seattle-takeawayddddresources/" href="http://elegantcode.com/2008/04/30/altnet-seattle-takeawayddddresources/">http://elegantcode.com/2008/04/30/altnet-seattle-takeawayddddresources/</a></p> <p><a title="http://www.martinfowler.com/eaaDev/EventSourcing.html" href="http://www.martinfowler.com/eaaDev/EventSourcing.html">http://www.martinfowler.com/eaaDev/EventSourcing.html</a></p>
